# S6：格式化字符串

## 漏洞原理

C 语言格式化函数缺少对可变参数的有效检查以及其丰富的格式化符号，通过向格式化函数传入恶意格式化串来达到窥视进程堆栈布局、修改函数返回地址等重要参数，进而控制程序执行。

格式化字符串利用分为三部分：格式化函数 + 格式化字符串 + 后续参数（可选）。

**关键格式说明**（N 为十进制整数）：

- `%N$p`：以 16 进制格式输出位于格式化函数第 N 个参数位置的值
- `%N$s`：以第 N 个参数位置的值为地址，输出地址指向的字符串内容
- `%N$n`：以第 N 个参数位置的值为地址，将已输出字符数量写入该地址（`%n` 写 4 字节、`%hn` 写 2 字节、`%hhn` 写 1 字节）
- `%Nc`：输出 N 个字符，配合 `%N$n` 可实现任意地址任意值写入

---

## 涉及函数清单

输出类：
- `printf` / `fprintf` / `sprintf` / `snprintf`
- `vprintf` / `vfprintf` / `vsprintf` / `vsnprintf`
- 华为内部：`VOS_nsprintf` / `VOS_nvsprintf` / `VOS_snprintf` / `snprintf_safe`

输入类：
- `scanf` / `fscanf` / `sscanf` / `vsscanf`

---

## 三种漏洞模式

### 模式 1：格式化串由外界提供


```c
// 危险：agname 来自外部网络报文
vsnprintf(buf, size, agname, args);
```



格式化串必须为**编译期常量字符串字面量**或**可信来源的常量**。任何来自网络、文件、
用户输入的数据都不得直接作为格式化串。

### 模式 2：格式化字符串要求的参数个数大于实际传入


```c
printf("%d %s %x", a);                    // 要求 3 个参数，实际 1 个
```



多余的 `%` 说明符会从栈上任意位置读取数据，造成栈信息泄漏，严重时 `%s` 对非法地址解引用导致崩溃。

### 模式 3：误用 snprintf/vsnprintf 等函数的返回值


```c
j += snprintf_safe(Buff + j, remain, "%s", long_string);
// 当 long_string > remain 时，实际写入被截断，但返回值仍为期望长度
// j 累加后超过 Buff 真实大小
j += snprintf_safe(Buff + j, remain2, "%s", "...");   // Buff+j 已越界
```



关键：C 标准 `snprintf` 返回值是**如果缓冲区足够大将写入的字符数**，不是实际写入数。
累加使用时必须用 `MIN(ret, remain)` 或重新计算剩余空间。

---

## 审计流程

### 1. 识别格式化函数调用

扫描函数体内上述函数的调用。

### 2. 模式 1 检查：格式化串是否为常量字面量

查看格式化串参数：
- 是字符串字面量（如 `"abc %d"`） → 排除模式 1
- 是变量、拼接结果、来自参数/结构体/全局 → **高危**，追踪来源：
  - 来自当前函数参数 → 根据 candidate 描述中的调用链线索看调用方是否传入常量
  - 来自外部数据 → **确认漏洞**

### 3. 模式 2 检查：参数个数匹配

对字面量格式化串：
- 数出 `%` 说明符个数（排除 `%%`）
- 对照实际传入参数个数
- 不匹配 → **高危**（参数少于说明符时读栈）

### 4. 模式 3 检查：返回值累加模式

搜索 `j += snprintf(...)` / `pos += snprintf(...)` 等累加形式：
- 累加前是否限制 size 参数随累加变小（`BUF_SIZE - j`）
- 累加后是否有 `if (j >= BUF_SIZE) break` 之类上界检查
- 返回值是否用 `MIN(ret, remain)` 处理

同时对 `snprintf` 本身应用 S1 分析（若它是 `_s` 变体或需要 buf+j 不越界）。

### 5. 输出缓冲区 vs 输入字符串长度检查

对 `sprintf`（无长度保护）：
- 目标缓冲区大小 vs 格式化结果最大可能长度
- 格式化结果含 `%s` 来自外部 → 长度不可控 → **确认漏洞**

---

## 判定标准

| 情况 | confirmed |
|------|-----------|
| 格式化串为变量，来源不可追溯为编译期常量 | `true` |
| 格式化串来自外部输入（网络/文件/参数） | `true` |
| 字面量格式化串的说明符数 > 实际参数数 | `true` |
| `sprintf` 存在 `%s` 参数来自外部输入且无长度保护 | `true` |
| 累加 `j += snprintf(...)` 无后续 j 上界截断 | `true` |
| `sscanf` 的 `%s` 无宽度限定符 | `true` |
| 格式化串为编译期常量且说明符与参数匹配 | `false` |

---

## 与其他场景的级联

- **S6 + S1**：`snprintf_s` 的 destSize 错误 → 同时属于 S1
- **S6 + S4**：`printf("%s", p)` 中 p 无 `\0` 终止 → 读越界，记 S4
- **S6 + S2**：格式化长度参数存在算术溢出

## 案例参考

**案例一（模式 1）：** 从外部网络报文获取 agname 信息，直接作为 vsnprintf 格式化串，
攻击者构造含 `%n` 的串实现任意地址写入。

**案例二（模式 2）：** 格式化字符串要求输入字符个数分别为 1, 7, 2，实际传入参数个数
分别为 0, 6, 1，造成栈信息泄漏。

**案例三（模式 3）：**


```c
j += snprintf_safe(Buff + j, remain, fmt1, long_input);     // 截断但 j 按期望长度累加
j += snprintf_safe(Buff + j, remain2, "%s", "...");          // Buff+j 已越界
```

## ai_analysis 输出模板


```
场景：S6 格式化字符串（模式 1：格式化串非常量）
代码：vsnprintf(szBuf, sizeof(szBuf), agname, args)
关键变量：格式化串 agname 来自外部网络报文（pMsg->agname）。
校验情况：未对 agname 内容做任何过滤。
判定：攻击者可构造含 %n 的 agname 实现任意地址写入任意内容。
修复建议：格式化串必须使用编译期常量，改为 vsnprintf(szBuf, sizeof(szBuf), "%s", agname)。
```




```
场景：S6 格式化字符串（模式 3：snprintf 返回值误用）
代码：j += snprintf_safe(Buff + j, MAX - j, "%s", pInput);
关键变量：Buff 大小为 MAX；j 按 snprintf 返回值累加。
校验情况：累加后无 j < MAX 的截断。
判定：pInput 较长导致返回值 > MAX-j，j 累加后 Buff+j 越界。
修复建议：累加时使用 j += MIN(ret, MAX - j - 1) 或在累加后校验 j。
```
