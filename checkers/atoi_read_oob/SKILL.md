# atoi 参数读越界审计

你正在分析一个 C/C++ `atoi` 调用点。静态分析已经枚举出调用位置，你需要判断该处是否存在真实的读越界风险。

`atoi` 会把参数当作 NUL 终止的 C 字符串读取，跳过前导空白并解析可选符号和数字，直到遇到非数字或 `\0`。需要分析传给 `atoi` 的指针从起始位置开始，是否可能在有效对象边界内找不到终止条件，导致继续读到边界之外。

## 输出要求

审计完成后必须调用一次 `submit_result`，参数规则：

- `confirmed`: 存在可达读越界路径时为 `true`，否则为 `false`
- `severity`: 外部输入可触发且可能跨对象读取为 `"high"`；内部数据流但可达为 `"medium"`；边界依赖较弱为 `"low"`
- `description`: 一句话说明本次 `atoi` 调用是否可能读越界
- `ai_analysis`: 按下面模板写清楚证据

`ai_analysis` 模板：

```text
调用点：<file:line function>
代码：<atoi 调用表达式>
参数来源：<参数表达式及来源>
边界与终止符：<对象大小/剩余空间/NUL 终止保证>
现有校验：<已有检查或“未发现”>
判定：<为什么确认读越界或为什么安全>
修复建议：<如改用 strtol 并先做长度内 NUL 检查/复制到带终止符缓冲区/补零>
```
